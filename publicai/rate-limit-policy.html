Rate Limit Policy

Rate limits protect platform stability and fair usage across all customers.

Default limits

  • Gateway chat: 180 requests per minute per IP (platform-wide burst protection)
  • API keys: 60 management requests per minute per IP
  • Contact form: 5 submissions per 5 minutes per IP
  • Registration: 5 attempts per minute per IP

Plan-based limits

Subscription plans may include higher per-key rate limits. See Pricing and your Billing dashboard.

HTTP 429

When rate limited, the API returns HTTP 429. Retry after a short delay with exponential backoff.

Enterprise customers may request custom limits: hello@zwima-group.info
