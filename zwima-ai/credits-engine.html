Zwima Group Online · Billing Logic

Zwima Credits Engine

Internal billing model for converting provider API costs into Zwima Credits, margins, and customer-facing packages.

1 Credit = internal billing unit. Credits translate provider model usage into a simple customer-facing balance while preserving margin control across different models.

Credit Definition

1 Credit = internal billing unit. Credits normalize different provider prices into one Zwima billing system. Customers see credits; internally Zwima maps those credits to real provider token costs and target margins.

Unified formula: 1 Credit = internal unit. Model Cost = 80% input token cost + 20% output token cost. Credits Cost = Model Cost / Zwima Credits. Customer Price = Credits Cost × target multiplier.

Model Cost is per 1M blended tokens. Gross margin is revenue-basis margin: (Customer Price - Credits Cost) / Customer Price.

Formulas

Blended API Cost

blended_cost = input_price × 80% + output_price × 20%

Credits Cost

credits_cost = model_cost / zwima_credits

Customer Price + Margin

customer_price = credits_cost × multiplier gross_margin = (customer_price - credits_cost) / customer_price

Pricing Recommendations

Recommended customer credit prices should be based on the model access tier. This prevents cheap models from subsidizing premium reasoning models.

Budget Credits

$0.40 / credit For Qwen Turbo, GPT-4o Mini, DeepSeek V3/R1, and other low-cost routing. 300%+ target

Standard Credits

$1.00 / credit Default customer-facing credit price for mixed model access. 300% on high-cost average

Premium Credits

$1.75 / credit For unrestricted access to GPT-4o, GPT-4.1, Claude Opus, Gemini 2.5 Pro, and premium routing. 500% target
